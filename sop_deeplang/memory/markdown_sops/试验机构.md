### **第一部分：核心准则**
在报告该章节时，必须严格使用提供的固定公司信息作为静态文本，直接复制验证方案中的名称、地址、邮编和电话等内容。无动态数据输入时，采用声明式表述确认信息完整性，不添加任何额外解释或数据。确保输出格式为表格化结构，保持商务正式语气。

### **第二部分：通用模板**
使用以下脚本指令生成报告章节：

1. 从验证方案提取静态信息，包括公司名称 `{{Company_Name}}`、地址 `{{Address_Line_1}}` 和 `{{Address_Line_2}}`、邮编 `{{Postal_Code}}`、电话 `{{Phone_Number}}`。
2. 如果无动态数据，插入声明：“--- GLP 报告 (Report) --- 无数据”。
3. 以表格格式输出：

| 名称： | {{Company_Name}} |
| --- | --- |
|   | {{Address_Line_1}} |
|   | {{Address_Line_2}} |
| 邮编： | {{Postal_Code}} |
| 电话： | {{Phone_Number}} |
|   |   |
