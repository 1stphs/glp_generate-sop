第一部分：核心准则
- 必填字段（必须逐项核对并记录来源）：
  1. 名称（中文全称） -> 动态数据
  2. 英文名称（如存在） -> 动态数据
  3. 地址（按行拆分：地址行1、地址行2…） -> 动态数据
  4. 联系方式（含国家区号） -> 动态数据
  5. 邮编 -> 动态数据
- 静态文本 vs 动态数据原则：章节标签（如“名称：”等）视为静态文本，字段值为动态数据。输出模板中不得嵌入具体数值，所有动态项使用占位符。
- 来源优先级与差异处理：
  1. 若报告(VR)中存在对应字段，优先采用VR值并记录VR页码/段落引用；同时从方案(VP)比对并记录差异。
  2. 若VR缺失，则采用VP并标注“仅在VP中发现”。
  3. 若VP与VR不一致，需在差异表中列出字段名、VP值、VR值、首选值与处理建议（采纳/需澄清），并将原始摘录一并保存以备审计。
- 可变格式与保真规则：
  1. 原始文本必须逐字保存（包含标点、换行），并同时提供必要的标准化展示（见模板）。
  2. 若存在中英文并列，应同时记录两种文字并在最终输出中并列显示。
- 可审计性与记录：
  1. 每次提取须记录提取者、提取时间、数据来源文件名与页码/段落。
  2. 若后续人工核实/变更，须追加修订记录（操作者、时间、理由）。
- 缺失阻断规则：若VP与VR均无该章节任何实质性数据，必须仅输出：“该章节在原始材料中无明确数据支持”。

第二部分：通用模板（可由AI执行的脚本指令）
1. 输入定位
   - 读取文件：VP_path，VR_path
   - 在VR_path中定位章节标题/关键词（如“名称”、“地址”、“联系方式”、“邮编”）
2. 提取字段（逐字段操作）
   - 对于每一字段 F ∈ {Facility_Name_CN, Facility_Name_EN, Address_Line1, Address_Line2, Contact, Postal_Code}：
     a. 在VR中搜索对应标签及其后文本，若找到则赋值 VR_F，同时记录 VR_source_location（文件名:页码/段落）。
     b. 在VP中同样搜索并赋值 VP_F（若存在），记录 VP_source_location。
3. 差异比对
   - 若 VR_F 存在，Preferred_F := VR_F；否则 Preferred_F := VP_F。
   - 若 VR_F 与 VP_F 均存在且 VR_F != VP_F，则在差异表中生成记录：
     {Field: F, VP_value: VP_F, VR_value: VR_F, Preferred_value: Preferred_F, VP_location, VR_location, Action: "Flag for QA/Clarification"}
4. 输出构建（占位符形式，不含具体数据）
   - 构建最终段落模板（用于报告/合并）：
     名称：{{Facility_Name_CN}}（{{Facility_Name_EN}}）
     地址：{{Address_Line1}}
           {{Address_Line2}}
     联系方式：{{Contact}}
     邮编：{{Postal_Code}}
   - 同时输出原始摘录区块（逐字段原文）和差异表（若存在差异）。
5. 审计元数据
   - 在输出文件尾部附加：Extraction_By={{Operator}}, Extraction_Time={{YYYY-MM-DD hh:mm:ss}}, VP_Source={{VP_path:location}}, VR_Source={{VR_path:location}}。
6. 分支逻辑
   - 若对任一必填字段均未在VP与VR中找到，停止并输出单行：该章节在原始材料中无明确数据支持
7. 交付物
   - deliverables：{Formatted_Block, Original_Snippets, Difference_Table (if any), Audit_Metadata}
8. 输出格式要求
   - 所有占位符以 {{Field_Name}} 形式出现；不得在模板中填充真实数值；对外最终报告段落可替换占位符为Preferred_F值。
