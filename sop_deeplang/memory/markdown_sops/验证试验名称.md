第一部分：核心准则
- 定义：本章节仅针对“验证试验名称（Study/Validation Title）”项的核对与记录，必须以协议（Protocol, VP）为准，报告（Report, VR）中的标题为静态文本复制产出，任何变更须有可追溯的理由与签署批准。
- 二元差异定位：明确区分
  - 静态文本（Static）：试验名称、方法名称、物种、样本类型、分析物描述等，应直接从VP复制到VR或记录变更原因。占位符示例：{{Method}}, {{Analyte}}, {{Species}}, {{Matrix}}, {{Form}}。
  - 动态数据（Dynamic）：本章节通常不包含数值型动态数据；如存在（例如并列的序号、日期内嵌在标题中），应从原始记录提取并在Delta记录中注明来源。
- 字符串规范化（比对前必须逐步执行，原子化描述）：
  1. 从源文本移除所有HTML标签（保留文本内容）。
  2. 解码HTML实体（例如 &amp; -> &）。
  3. Unicode正规化至NFKC。
  4. 去除首尾空白；将连续空白合并为单一空格。
  5. 比较时区分或不区分大小写按业务规则（默认区分大小写，若VP有说明则遵从）。
- 合规要求：
  - 若VR标题与VP完全一致（经上述正规化后），判定为“Match”；记录核对人、核对时间：{{Reviewer}}, {{Review_Date}}。
  - 若不一致，必须在差异记录（Delta）中包含：VP原文、VR原文、正规化后文本、差异类型（Missing/Modified/Formatting）、变更理由（必填）与审批签章（{{Reason_Code}}, {{Approver}}, {{Approval_Date}}）。
- 空数据阻断：若VP和VR均无本章节实质内容，必须输出单独一行：
  该章节在原始材料中无明确数据支持
  （并停止后续自动化写入本章节内容）。

第二部分：通用模板
- 输入定位：
  1. 提取VP标题：从文件路径 {{VP_File_Path}} 与定位器 {{VP_Title_Path}}（例如：Protocol -> Title）读取原文并存为 {{VP_Title_Raw}}。
  2. 提取VR标题：从文件路径 {{VR_File_Path}} 与定位器 {{VR_Title_Path}}（例如：Report -> Title 或 报告首页）读取原文并存为 {{VR_Title_Raw}}。
- 正规化函数（执行顺序，伪代码风格）：
  1. function Normalize(text):
       a. text1 = StripHTMLTags(text)
       b. text2 = DecodeHTMLEntities(text1)
       c. text3 = UnicodeNormalizeNFKC(text2)
       d. text4 = Trim(text3)
       e. text5 = CollapseWhitespace(text4)
       f. return text5
  2. {{VP_Title_Norm}} = Normalize({{VP_Title_Raw}})
     {{VR_Title_Norm}} = Normalize({{VR_Title_Raw}})
- 比对逻辑：
  1. if {{VP_Title_Norm}} == {{VR_Title_Norm}} then
       - result = "Match"
       - produce record: {Field:"Title", VP_raw:{{VP_Title_Raw}}, VR_raw:{{VR_Title_Raw}}, VP_norm:{{VP_Title_Norm}}, VR_norm:{{VR_Title_Norm}}, Status:result, Reviewer:{{Reviewer}}, Review_Date:{{Review_Date}}}
  2. else
       - classify difference:
         a. If VR empty -> Status="Missing_in_VR"
         b. If VP empty -> Status="Missing_in_VP"
         c. Else -> Status="Modified"
       - require filling: Reason ({{Reason_Text}}), Impact_Assessment ({{Impact}}), Approver ({{Approver}}), Approval_Date ({{Approval_Date}})
       - produce Delta record with all fields above and attach screenshots/paragraph excerpts.
- 输出与记录位置：
  - 将比对记录保存到指定目录/表单：{{Delta_Log_Path}}，命名规则：{{StudyID}}_Title_Check_{{YYYYMMDD}}.json（或xlsx），并将所用源文件路径写入记录。
- 分支处理（特殊情况）：
  - 若存在多语言版本，分别对比每一语言版本并在记录中列明语言标签{{Lang}}。
  - 若标题包含嵌入日期/编号作为动态字段：在比较前先按正则提取并用占位符替换（示例：ReplaceDates -> {{Date_Placeholder}}），并在记录中保存原始值。
- 自动化执行提示（供下游AI使用）：
  - 步骤序列：读VP -> 读VR -> Normalize -> Compare -> If mismatch collect Reason & route for Approval -> Save Delta -> Insert final Title into VR稿件（若Status=Match直接保留VR原文；若Status=Modified并获批，用经批准的文本替换VR标题）。
  - 禁止行为：不得基于上下文或推测自行改写标题；任何改写必须在Delta记录有理据与签名。
