第一部分：核心准则
- 定义：本章节关于QA检查/核查记录的撰写必须严格遵守下列“法律”条目。
  1. 必须二元区分并显式记录：静态文本（直接来自验证方案的文字）与动态数据（需从原始记录/审核单据中提取的字段）。静态文本在报告中可复述；动态数据必须引用来源并保留可追溯性（附件编号/页码/文件名）。
  2. 检查项目必须按“检查类型 / 检查项目 / 检查日期 / 报告至管理者日期”四列结构化呈现；每一单元格为动态字段并注明数据来源。
  3. 若有例外（例如：缺失某项原始证明文件如CoA），必须在QAU声明中明确披露并注明被排除的具体文件名称，且不得对该缺失项进行主观推断。
  4. QAU结论性陈述必须包含：验证试验编号、验证试验名称、QAU对报告反映原始数据准确性的总体意见（有无例外），以及异常项的具体列举与来源引用。若为双语报告，中文与英文内容应语义对应。
  5. 签名信息（签名人中文名、拼音、职称、职位）与职务（QA检查员）必须作为固定静态段落出现，且该段落内的个人信息为动态字段。
  6. 所有日期字段应保持原始格式并在模板中以占位符表示（禁止硬编码具体日期）。
  7. 若原始材料无相关检查记录，必须返回规定短句（见结构强制对齐第3部分分支逻辑）。

第二部分：通用模板
- 定义：供AI Generator执行的逐步提取与填充脚本指令。所有模板字段均为占位符，不含具体数值。
1) 元数据与声明段（静态/动态标注）
   - 写入：Validation Study Number: {{Validation_Study_Number}}  
   - 写入：Validation Study Title: {{Validation_Study_Title}}  
   - 写入QAU声明（双语占位）：  
     - 中文：{{QAU_Chinese_Statement}}  
     - English：{{QAU_English_Statement}}  
   - 来源指引：从验证报告正文“QAU检查”段提取上述字段；若原文仅为中文或仅为英文，按原文填充相应占位符并在另一语言占位符标注“NA”。

2) 检查表提取规则（强制四列）
   - 对报告中出现的每一行检查记录，提取并填充以下字段：  
     - {{Inspection_Type}}  
     - {{Phases_Inspected}}  
     - {{Inspection_Date}}  
     - {{Reported_to_VSD_or_TFM_Date}}  
   - 格式化输出为表格或对齐文本，保持原始日期字符串不变。每条记录后追加来源引用：({{Source_File}}; page {{Page_Number}})。

3) 例外与披露
   - 检查CoA或其他证明文件是否在原始材料中缺失：  
     - 若存在缺失，将单独生成条目：Exception: {{Missing_Document_Name}} — described as excluded from audit, source: {{Source_File}}。  
     - 同时在QAU声明中追加：“(except {{Missing_Document_List}})”占位符。

4) 签名与岗位
   - 提取并填充：{{QAU_Chinese_Name}} ({{QAU_Pinyin}}), {{QAU_Title}}  
   - 强制写入职位文本：QA检查员（Quality Assurance Auditor）  
   - 若报告中无签名行，保留占位符并在来源栏注明“签名/职称在原件缺失”。

5) 输出合成规则（顺序与最小内容）
   - 输出顺序固定：验证试验编号 -> 验证试验名称 -> QAU声明 -> 检查表 -> 设施基于SOP的检查声明（若有） -> 签名信息 -> 例外条目（若有）。  
   - 若原始材料中未找到任何与本章节对应的检查数据，程序必须输出精确句子：该章节在原始材料中无明确数据支持，并终止后续填充步骤。

6) 质量与追溯性注记
   - 每次填充后，生成字段映射清单：{{Field_Name}} := extracted from {{Source_File}} (page {{Page_Number}})。该映射随报告一并保存以便审计追溯。
