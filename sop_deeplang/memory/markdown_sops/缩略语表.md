第一部分：核心准则
- 目的：生成一份与验证方案(Protocol)和GLP报告(Report)一致的“缩写表（Abbreviations）”标准化输出，明确每条目来源并记录差异。
- 静/动二元划分（必守）：
  - 静态文本（Static）：在Protocol中出现且与Report文本一致的缩写条目，直接采用Protocol文本作最终呈现。
  - 动态数据（Dynamic）：仅在Report中出现或Report与Protocol在“全称/中文含义”上不一致的条目，视为动态，必须从原始记录提取并标注来源。
- 冲突处理（必守）：
  - 若Protocol与Report条目不一致，禁止任意选择任一表述作为最终文本。必须：
    1) 在输出中同时列出两方表述（字段：Protocol_Value / Report_Value），
    2) 将该条目标记为“冲突（CONFLICT）”，
    3) 生成供QA复核的差异记录（字段：Discrepancy_Record_Path = {{Discrepancy_File_Path}}），并在最终表中以占位符 {{QA_Resolution}} 标注待决结论。
- 格式与字段要求（必守）：
  - 最终标准化表每条必须包含且仅包含下列字段（列顺序固定）：Abbreviation（缩写） | English_Full_Name（英文全称） | Chinese_Meaning（中文含义） | Source（来源：Protocol/Report） | Status（STATIC/DYNAMIC/CONFLICT） | Notes（如有）
  - 字符规范：Abbreviation 全大写；English_Full_Name 首字母大写按原文；Chinese_Meaning 使用报告提供的中文表述，不做译改。
- 不可制造/改写数据（必守）：
  - 严禁为缺失条目编造英文全称或中文含义；若两处均缺失，则在输出中列为“缺失（MISSING）”并标注来源与校核动作。
- 元数据记录（必守）：
  - 输出必须包含数据来源元信息：Protocol 文件名/版本/段落标识（{{Protocol_File}}/{{Protocol_Version}}/{{Protocol_Section}}）、Report 文件名/版本/段落标识（{{Report_File}}/{{Report_Version}}/{{Report_Section}}）、提取时间戳（{{Extraction_Timestamp}}）。

第二部分：通用模板
- 输入：
  - {{Protocol_File}}（路径或附件标识）
  - {{Report_File}}（路径或附件标识）
- 输出：
  - 标准化缩写表：{{Output_File_Abbrev_Table}}（CSV/TSV/JSON 任一，默认 CSV 列顺序同“格式与字段要求”）
  - 差异清单：{{Discrepancy_File_Path}}
- 步骤（可直接作为自动化脚本指令执行）：
  1) 从 {{Protocol_File}} 中定位“缩写/术语表”段落或表格：
     - 如存在结构化表格，直接抽取列：Abbreviation、English_Full_Name、Chinese_Meaning；
     - 如为行文本，按行解析并匹配模式：首字段为大写缩写并随后包含全称与中文含义。
     - 若未找到，记录 Protocol_Source_Exists = false。
  2) 从 {{Report_File}} 中同样抽取缩写表（同上逻辑）；若未找到，记录 Report_Source_Exists = false。
  3) 标准化两侧提取结果：
     - 修剪空格、统一空格、移除尾随标点（不改变原文词序或大小写规则）。
     - 确保 Abbreviation 全大写；若不符合则保留原文并在 Notes 标注“原文大小写异常”。
  4) 比对（按 Abbreviation 精确匹配）：
     - 若 Protocol 存在且 Report 存在且 English_Full_Name 与 Chinese_Meaning 完全一致 → 标记 Status=STATIC，Source=Protocol。
     - 若 Protocol 存在且 Report 缺失 → 将 Protocol 条目拷贝为最终条目，Status=STATIC，Source=Protocol（Notes: Report_missing）。
     - 若 Protocol 缺失且 Report 存在 → 将 Report 条目拷贝为最终条目，Status=DYNAMIC，Source=Report。
     - 若双方均存在但任一字段不同（英文全称或中文含义）→ Status=CONFLICT，Source=Protocol/Report，并在差异清单中记录两侧原文（字段：Protocol_Value, Report_Value）。
  5) 生成最终表格（{{Output_File_Abbrev_Table}}）：
     - 遵循“格式与字段要求”列顺序；
     - 对 Status=CONFLICT 的行，在 Notes 填入差异摘要并写入 Discrepancy_File_Path；
     - 包含文件级元数据头（Protocol/Report 文件名与版本、提取时间）。
  6) 若存在任何 MISSING 或 CONFLICT 条目，自动生成待QA复核的差异报告（{{Discrepancy_File_Path}}），并在报告顶部写入下一步动作建议占位符（{{QA_Action}}）。
- 异常与边界处理：
  - 若两文件均未包含缩写段落 → 终止并返回文本：该章节在原始材料中无明确数据支持（严格执行强制空数据阻断）。
  - 若同一缩写在同一文件中多次出现且定义不同 → 视为内部冲突，标记为 CONFLICT 并列出所有变体供QA判定。
- 输出示例列头（CSV）：
  - Abbreviation,English_Full_Name,Chinese_Meaning,Source,Status,Notes
