### **第一部分：核心准则**
* 本章节撰写必须严格遵守 GLP 法规（21 CFR Part 58）及 FDA 21 CFR Part 11 电子记录要求，确保所有数据描述基于原始记录提取，区分静态文本（直接引用协议接受标准）与动态数据（实际测试结果）。
* 差异定位：协议中的接受标准（如限值范围）为静态红线，直接复制；报告中的实际数据（如日期、结果值、判定）为动态变量，必须从原始记录（如附件表格）提取并计算。
* 计算逻辑：若涉及统计（如均值、CV%），需原子化步骤：1) 提取所有重复值；2) 计算均值（保留1位小数）；3) 计算 CV% = (标准差/均值)×100（保留1位小数）；4) 与协议限值对比，判定“符合”或“不符合”。
* 分支逻辑：若实际结果 ≤ 协议接受标准上限且 ≥ 下限，则判定“符合”；否则，报告“不符合”并追溯原因（e.g., "超出限值 X%"）。
* 通用化原则：禁止硬编码具体数值，使用占位符如 {{Test_Date}}、{{Mean_Value}}、{{CV_Percent}}；确保逻辑闭环，支持不符合情况描述。

### **第二部分：通用模板**
```
【执行指令】：
1. 从原始记录/附件 {{Attachment_ID}} 提取测试日期 {{Test_Date}}、重复测量值列表，并计算：
   - 均值 {{Mean_Value}} = AVERAGE(重复值列表)，保留 {{Decimal_Places}} 位小数。
   - CV% {{CV_Percent}} = (STDEV(重复值列表)/{{Mean_Value}}) × 100，保留 {{Decimal_Places}} 位小数。
2. 引用协议接受标准：{{Protocol_Criteria}}（静态复制）。
3. 对比判定：
   - 若 {{CV_Percent}} ≤ {{Acceptance_Limit}}，判定“所有结果符合接受标准”。
   - 否则，判定“不符合，实际 {{CV_Percent}} 超出限值 {{Excess_Percent}}”。
4. 填入以下模板，保持 GLP 客观语气，确保数据可追溯至原始记录。

【填充模板】：
测试于 {{Test_Date}} 进行 {{Replicates_Number}} 次重复测量，结果分别为 {{Result_1}}、{{Result_2}}、...、{{Result_N}}。计算得均值为 {{Mean_Value}}，CV 为 {{CV_Percent}}%。协议接受标准为 {{Protocol_Criteria}}。所有结果[判定：符合/不符合]该标准。
```

### **第三部分：报告示例**
测试于 2025-01-15 进行 6 次重复测量，结果分别为 98.2%、99.1%、97.8%、98.5%、99.0%、98.3%。计算得均值为 98.5%，CV 为 0.6%。协议接受标准为差异 ≤ ±15.0%。所有结果符合该标准。
