### 第一部分：核心准则
* **定义**：该章节撰写时必须死守的"法律"。
  - 严格记录验证日程的关键日期节点，包括方案批准日期、实验开始日期、实验结束日期及报告终稿完成时间，确保所有日期格式统一为“YYYY年MM月DD日”。
  - 日期信息直接从实验记录或批准文档中提取，不得推断或修改；若无具体数据，默认使用占位符并注明“待填”。
  - 所有日期必须可追溯至原始记录，确保时间线逻辑闭环（批准日期 ≤ 开始日期 < 结束日期 < 报告完成日期）。
  - 修约规则：日期无统计计算需求，直接使用原始记录值，不涉及小数修约；若涉及持续天数计算（如实验总时长），则采用四舍五入至整数天，保留原位数。

### 第二部分：通用模板
* **定义**：可直接由 AI Generator 执行的脚本指令。
  ```
  从实验批准文档和记录日志中提取以下关键日期：
  - 方案批准日期：{{Approval_Date}}（格式：YYYY年MM月DD日）
  - 实验开始日期：{{Start_Date}}（格式：YYYY年MM月DD日）
  - 实验结束日期：{{End_Date}}（格式：YYYY年MM月DD日）
  - 报告终稿完成时间：{{Report_Completion_Date}}（格式：YYYY年MM月DD日）

  验证时间线逻辑：确保 {{Approval_Date}} ≤ {{Start_Date}} < {{End_Date}} < {{Report_Completion_Date}}；若不符合，标记为“需审核”。

  若实验总时长需计算：{{Total_Days}} = {{End_Date}} - {{Start_Date}}（以天为单位，四舍五入至整数）。

  输出表格格式：
  方案批准日期：	{{Approval_Date}}
  实验开始日期：	{{Start_Date}}
  实验结束日期：	{{End_Date}}
  报告终稿完成时间：	{{Report_Completion_Date}}

  若无偏离，直接陈述“验证日程按方案执行，无延误”；若有偏差，添加说明“实际结束日期延后{{Deviation_Days}}天，原因：{{Reason}}”。
  ```
