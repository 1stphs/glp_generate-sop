### **第一部分：核心准则**
在GLP报告的委托方信息章节中，必须严格按照验证方案(Protocol)中提供的静态信息进行直接复制和呈现，确保无任何数据偏差或遗漏。动态内容（如报告日期）应从报告上下文提取，但本章节无数据时，采用声明式表述确认信息完整性。无偏离情况下，表述为“信息符合方案要求，无偏差”。

### **第二部分：通用模板**
1. 从验证方案(Protocol)中提取委托方静态信息，包括名称、地址、邮编、电话。
2. 填入以下模板结构，确保格式对齐表格形式：
   | 名称： | {{Client_Name}} |
   | --- | --- |
   | 地址： | {{Client_Address}} |
   | 邮编： | {{Client_Postcode}} |
   | 电话： | {{Client_Phone}} |
3. 若无额外数据，添加声明：{{Declaration}}（例如：“信息符合方案要求，无偏差”）。
