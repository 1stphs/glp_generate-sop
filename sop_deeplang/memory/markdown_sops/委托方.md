### **第一部分：核心准则**
* 严格遵守 GLP 法规（21 CFR Part 58）和 FDA 21 CFR Part 11 电子记录要求，确保报告章节内容基于验证方案（VP）中的预定义接受标准与实际数据对比，客观记录事实，无主观臆断。
* 区分静态文本（直接引用 VP 中的标准描述，如"字符数限制"）与动态数据（从原始记录提取，如实际字符数）。
* 数据处理逻辑：字符数计算采用 `len(string.strip())` 方法（去除首尾空白）；若实际字符数超出 VP 阈值，客观报告事实并标注"不符合"，不省略任何不符合项。
* 通用化原则：禁止使用具体数值（如"66"、"70"），一律用占位符 `{{VP_Threshold_Chars}}` 和 `{{Actual_Chars}}`。
* 分支逻辑：符合 → "满足接受标准"；不符合 → "超出接受标准，实际为{{Actual_Chars}}"；边缘情况（如相等）视为符合。
* 合规表述：使用过去时态描述实际结果（如"测得"、"记录为"），并引用数据来源（如"根据附件X计算"）。

### **第二部分：通用模板**
```
【章节撰写指令】：
1. 从 VP 中提取静态标准：协议最大字符数限制为 {{VP_Threshold_Chars}} 字符。
2. 从原始记录/附件中提取动态数据：实际协议字符数 {{Actual_Chars}}（计算方法：len(协议文本.strip())）。
3. 执行对比：
   - 若 {{Actual_Chars}} ≤ {{VP_Threshold_Chars}}，判定为"符合"。
   - 若 {{Actual_Chars}} > {{VP_Threshold_Chars}}，判定为"不符合"。
4. 填充以下报告模板，确保 FDA Part 11 合规（电子签名可追溯）：

"协议内容字符数验证：
根据验证方案（VP），协议最大字符数限制为 {{VP_Threshold_Chars}} 字符。
实际测得协议字符数为 {{Actual_Chars}} 字符（详见附件 [数据来源引用]）。
[判定结果：{{符合/不符合}} 接受标准]。"

数据流转路径：VP标准 → 原始文本提取 → Python len()计算 → 对比 → 模板填充。
```

### **第三部分：报告示例**
协议内容字符数验证：  
根据验证方案（VP），协议最大字符数限制为 66 字符。  
实际测得协议字符数为 70 字符（详见附件 原始协议文档）。  
不符合 接受标准。
