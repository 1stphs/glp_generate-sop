第一部分：核心准则
- 定义：本章节固定记录“委托方信息”，必须从原始验证方案(Protocol)中提取并与报告草稿(Report)核对，最终在报告中以一致的、经过审批的文本呈现。
- 必填字段：{{Client_Name}}；{{Client_Address}}；{{Client_PostalCode}}。不得添加未经来源支撑的字段或值。
- 静态/动态区分：上述三项均属静态文本（直接复制来源）；不进行任何基于测量或计算的衍生数值。
- 规范化规则（用于比对与填充）：
  1. 去首尾空白并将连续空格合并为单个空格。
  2. 将全角标点与半角标点统一为中文标准（如“，”、“。”）。
  3. 保持原文字符顺序与大小写（中文无需大小写转换），比较为完全字符串相等（after normalization）。
- 差异处置：
  1. 若Protocol与Report中对应字段一致：记录来源文件名/页码，并直接采用该文本作为最终报告内容。
  2. 若不一致：在输出处标注为“需委托方确认”，记录两份原文、来源路径及差异类型（增补/删减/拼写不同），并将该记录提交给质量负责人（或委托方确认人）审批后再更新报告。
- 审计痕迹：每次提取/比对/变更必须记录操作人、操作时间、来源文件名与页码、变更前后文本、审批人及审批时间，符合可追溯性要求。
- 禁止行为：不得根据其他文档或经验猜测、合并或替换委托方信息。任何人工校正须有委托方或质量负责人书面确认。

第二部分：通用模板
- 输入：Protocol 文件路径 -> 定位到“委托方”段落；Report 草稿文件路径 -> 定位“委托方”段落。
- 输出变量：{{Client_Name}}；{{Client_Address}}；{{Client_PostalCode}}；{{Source_Protocol_File}}；{{Source_Report_File}}；{{Comparison_Result}}；{{Discrepancy_Record_Link}}。
- 操作步骤（可直接交付给下游AI/脚本执行）：
  1. 从 {{Source_Protocol_File}} 中提取“委托方”段落，赋值到临时变量 p_Name、p_Address、p_PostalCode；按规范化规则处理。
  2. 从 {{Source_Report_File}} 中提取“委托方”段落，赋值到临时变量 r_Name、r_Address、r_PostalCode；按相同规范化规则处理。
  3. 对三对字段逐项比较（p_XXX == r_XXX）：
     - 若全部相等：设置 {{Client_Name}}=p_Name；{{Client_Address}}=p_Address；{{Client_PostalCode}}=p_PostalCode；{{Comparison_Result}}="一致"；在审计记录中写入来源与页码。
     - 若任一不等：生成差异记录包含：字段名、p_值、r_值、来源文件与页码；设置 {{Comparison_Result}}="不一致"；在报告草稿处以显著标记（例如：<<需委托方确认>>）替换该字段占位并链接到差异记录 {{Discrepancy_Record_Link}}。
  4. 生成最终报告段落时，只允许使用经确认的{{Client_XXX}}变量；若存在未确认差异，必须保留“需委托方确认”标注并阻断最终发布流程。
  5. 记录审计条目：操作人、操作时间、原始文本快照、比较结果、提交确认人、确认时间。
- 输出格式（用于自动填充报告模板）：
  - 委托方
    名称：{{Client_Name}}
    地址：{{Client_Address}}
    邮编：{{Client_PostalCode}}
  （若 {{Comparison_Result}} == "不一致" 则在相应行后追加文本：<<需委托方确认，差异记录：{{Discrepancy_Record_Link}}>>）
