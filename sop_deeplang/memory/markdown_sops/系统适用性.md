### 第一部分：核心准则
系统适用性章节旨在验证检测仪器在进样时的正常运转状态。撰写时必须遵守以下规则：  
1. 从表2_系统适用性中提取关键数据，包括检测批次编号、CV（%）、待测物保留时间、内标保留时间和峰面积比。  
2. 计算统计参数：CV（%）为变异系数，修约为小数点后一位（例如，0.35修约为0.4）；保留时间和峰面积比修约为小数点后一位或整数（根据数据精度，保留时间若为0.4则保持，峰面积比若为8则保持整数）。所有统计参数必须一致修约，避免报告中出现不一致位数。  
3. 评估合规性：检查所有批次数据是否符合预设接受标准（例如，CV（%）≤指定阈值，如2.0%；保留时间稳定，无异常波动；峰面积比在预期范围内）。若所有结果符合，则使用肯定性描述；若有不符合，需列出具体偏差并说明影响。  
4. 数据流转路径：从Excel表2提取原始数据 → 计算整体合规率 → 引用表2 → 描述总体结论。无数据时，默认声明“按验证方案考察，结果符合标准”。  
5. 分支逻辑：若数据不符合标准，添加警示语句，如“部分批次CV超过阈值，建议重新验证仪器状态”。

### 第二部分：通用模板
执行以下脚本指令生成系统适用性章节：  
1. 提取数据：从附件Excel表2_系统适用性中读取所有批次数据，包括{{Batch_Number}}、{{CV_Percent}}（修约至小数点后一位）、{{Analyte_RT}}（待测物保留时间，修约至小数点后一位）、{{IS_RT}}（内标保留时间，修约至小数点后一位）和{{Peak_Area_Ratio}}（峰面积比，修约至小数点后一位或整数）。  
2. 计算合规性：统计{{Total_Batches}}个批次中符合标准的{{Compliant_Count}}（例如，CV（%）≤{{Acceptance_Threshold_CV}}，保留时间波动≤{{RT_Threshold}}，峰面积比在{{PAR_Range}}内）。若{{Compliant_Rate}} = 100%，则结论为“所有结果符合”；否则，列出不符合批次{{Non_Compliant_Batches}}。  
3. 填充描述：  
“系统适用性按验证方案考察，测定结果见表2，所有{{Total_Batches}}个批次的{{CV_Percent}}、{{Analyte_RT}}、{{IS_RT}}和{{Peak_Area_Ratio}}均符合接受标准（CV（%）≤{{Acceptance_Threshold_CV}}，保留时间稳定，峰面积比在{{PAR_Range}}内），表明进样时检测仪器均处于正常运转状态。”  
若不符合：  
“系统适用性按验证方案考察，测定结果见表2，{{Compliant_Count}}/{{Total_Batches}}个批次符合接受标准，{{Non_Compliant_Batches}}存在偏差（例如，批次{{Batch_Number}}的CV（%）={{CV_Percent}}超过阈值），建议检查仪器状态。”  
4. 排版：引用“表2：系统适用性”，确保数据表格完整嵌入报告。
