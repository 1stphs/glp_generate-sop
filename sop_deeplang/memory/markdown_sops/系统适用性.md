### 第一部分：核心准则
系统适用性章节旨在确认检测仪器在进样时的正常运转状态，必须基于验证方案中的接受标准进行评估。核心合规性底线包括：  
- 从原始数据（如表2）提取所有相关测定结果，包括定性或定量指标（如响应值、峰面积等）。  
- 计算统计参数（如均值、标准差、RSD%），并按以下修约规则标准化：所有百分比值保留1位小数（例如，RSD% 修约至 5.2%）；绝对值保留2位小数（例如，响应值 修约至 123.45）；若数据为整数指标，直接保留原位数无修约。  
- 与验证方案中的接受标准逐项对比，若所有结果符合标准，则判定为“符合”；若存在不符合项，必须列出具体偏差并说明影响。  
- 表述必须客观、闭环，确保逻辑覆盖“数据提取 → 统计计算 → 标准对比 → 状态结论”，无数据时默认声明“按方案考察，无偏离”。

### 第二部分：通用模板
执行以下脚本指令生成系统适用性章节：  
1. 从附件原始记录（如表2）提取所有系统适用性测定数据，包括{{Sample_ID}}、{{Measured_Value}}（如峰面积或响应）、{{ replicates}}（重复次数）。  
2. 计算统计参数：  
   - 均值 Mean = (Σ {{Measured_Value}}) / {{replicates}}，修约至2位小数。  
   - 相对标准差 RSD% = (标准差 / Mean) × 100，修约至1位小数。  
   - 其他参数（如线性范围、LOD/LOQ）若适用，按方案标准计算并修约一致。  
3. 对比验证方案接受标准：检查 {{Calculated_RSD}} ≤ {{Acceptance_Limit}}（例如 ≤ {{RSD_Max}}%），所有 {{Test_Items}} 是否均符合。  
4. 填充报告语句：  
   “系统适用性按验证方案考察，测定结果见表{{Table_No}}，所有结果均{{Compliance_Status}}接受标准，表明进样时检测仪器均处于正常运转状态。”  
   - 若不符合：替换为“部分结果不符合接受标准，具体见表{{Table_No}}，{{Deviation_Details}}，建议{{Remediation_Action}}。”  
   - 无数据时：使用“系统适用性按验证方案考察，无偏离，所有结果符合接受标准，检测仪器处于正常状态。”
