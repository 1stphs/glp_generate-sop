第一部分：核心准则
- 定义遵循源文本优先原则：将验证方案（Protocol）中“指导原则/参考文献”章节的文本视为静态权威来源；报告（Report）中相同章节应逐条复现协议中列出的条目，除非协议中无该条目（见第二条异常处理）。
- 二元差异定位：本章节仅包含“静态文本引用”（Guidance references）；不存在需从原始记录提取的“动态数据”。任何比对、归一化仅限字符串层面（空白、HTML 标签、行尾符号可归一化），不得修改文献标题实质内容（含年份、缩写、标点）。
- 一致性检查为必须步骤：若协议与报告在条目文本、顺序或缺失上存在不一致，必须在SOP输出中产生明确的差异标识（{{Discrepancy_Flag}}），并指明以协议为准或需追溯确认的项（使用占位符，不在此处做决定）。
- 格式化规则：报告输出采用逐条段落呈现，每条一段（建议使用 <p>…</p> 或等价段落标记），保持协议中出现的原始书目文本和顺序。禁止在引用条目中追加解释性文字或注释。

第二部分：通用模板
- 目标：从 Protocol 提取指导性文献列表，校验并生成报告段落，输出差异标识。
- 输入占位符：
  - {{Protocol_Path}}：协议文件中“指导原则/参考文献”章节的位置或标识。
  - {{Report_Path}}：报告文件中同章节的位置或标识（用于比对）。
  - {{Guidance_List}}：从协议中抽取的按顺序的条目集合。
- 操作步骤（可直接由 AI 执行）：
  1. 从 {{Protocol_Path}} 提取原始文本块，按行或段落分割生成列表 {{Guidance_List}}，每项记为 {{Guidance_Item_n}}。
  2. 归一化每项：去除首尾空白、连续空格归一、删去多余 HTML 空行，但保留原始标点与年份字符串。
  3. 从 {{Report_Path}} 提取对应章节文本并同样归一化，生成 {{Report_Item_m}} 列表。
  4. 对比规则（逐项）：
     - 若存在相同文本（归一化后完全相等），则标记为 MATCH 并在输出中照搬协议文本。
     - 若协议项在报告中不存在，则在输出中插入该协议项并在元数据中设定 {{Discrepancy_Flag}} = "Protocol_only: {{Guidance_Item_n}}"
     - 若报告项存在但文本不等，则在元数据中设定 {{Discrepancy_Flag}} = "Text_Mismatch: Protocol={{Guidance_Item_n}} | Report={{Report_Item_m}}"
     - 若报告包含协议未列之项，则在元数据中设定 {{Discrepancy_Flag}} = "Report_only: {{Report_Item_m}}"
  5. 生成最终报告段落：按协议中原顺序输出每个 {{Guidance_Item_n}}，每条作为独立段落；在文档尾部插入差异汇总表占位（{{Discrepancy_Summary}}）。
- 输出占位与格式：
  - 段落模板：<p>{{Guidance_Item_n}}</p>
  - 差异占位：<p>差异标识：{{Discrepancy_Summary}}</p>
- 异常处理：
  - 若协议与报告均无任何条目（空内容），则停止并返回文本："该章节在原始材料中无明确数据支持"
  - 若协议为空但报告有条目，采用报告条目生成输出并在差异占位中标注 "Protocol_missing"。
