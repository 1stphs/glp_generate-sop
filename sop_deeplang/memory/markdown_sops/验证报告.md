第一部分：核心准则
- 必填字段（法律性底线）：每份章节记录必须至少包含：试验名称（中英文或原文）、验证试验编号、验证负责人（Study Director）、试验机构名称及联系方式、委托方名称及地址、报告签发日期、总页数。  
- 静态文本 vs 动态数据：试验名称、试验编号、委托方与试验机构的法定名称可视为“静态文本”（优先来自Protocol）；报告签发日期、总页数、联系电话等视为“动态数据”（以最终Report为准）。所有动态数据必须能追溯到原始文件或签发记录。  
- 来源优先级与不一致处理：当Protocol与Report中同一字段不一致时，以Report为最终呈现，且必须在变更/偏差记录中说明差异来源与批准人（保留电子/纸质证据）。  
- 格式与占位要求：供下游自动化填充的模板不得包含具体数值；必须使用占位符（如 {{Study_Title}}、{{Study_ID}}、{{Report_Sign_Date}}、{{Total_Pages}} 等）。  
- 多语言与换行：若Report以中英文并列或包含显式换行（如 HTML <br/>），在生成最终报告示例时保留原样；在数据库/表单记录中同时保存“原文”与“纯文本（去除HTML）”两个字段以便审计。  
- 验证与记录：抽取并填充后，必须进行一次三点核对（抽取源、填充模板、最终Report）并记录核对人、核对时间与结果；若字段缺失或不完整，触发缺项流程并在日志中记录。  

第二部分：通用模板
1) 数据抽取指令（执行顺序）
   - 从Protocol提取：{{Study_Title_Protocol}}, {{Study_ID_Protocol}}, {{Study_Director_Protocol}}, {{Sponsor_Name_Protocol}}, {{Sponsor_Address_Protocol}}, {{Facility_Name_Protocol}}, {{Facility_Address_Protocol}}。  
   - 从Report提取（优先用于最终填充）：{{Study_Title_Report}}, {{Study_ID_Report}}, {{Study_Director_Report}}, {{Sponsor_Name_Report}}, {{Sponsor_Address_Report}}, {{Facility_Name_Report}}, {{Facility_Address_Report}}, {{Facility_Phone_Report}}, {{Report_Sign_Date}}, {{Total_Pages}}。  
   - 比对步骤：对每一字段执行“Protocol vs Report”比对，记录比对结果（MATCH / MISMATCH），若MISMATCH，复制Report值至最终字段并在{{Discrepancy_Log}}中记录原因与批准人。  

2) 数据验证规则（自动化校验）
   - {{Study_ID}}：必须与Protocol记录完全一致或有经批准的变更单；格式校验为字母/数字/短横的组合（按机构惯例）。  
   - {{Report_Sign_Date}}：若Report使用本地格式（如“YYYY年MM月DD日”），同时记录ISO格式（{{Report_Sign_Date_ISO}} = YYYY-MM-DD）。  
   - {{Total_Pages}}：必须为正整数；校验方法：将生成的PDF/打印件页数与此字段比对。  
   - 地址/电话字段：保留原文并去HTML标签存纯文本；电话号码格式化为国际标准（若可获得）。  

3) 输出模板（供自动化写入最终报告/数据库）
   - Study Title: {{Study_Title}}  (若包含英文，格式：{{Study_Title_CN}}<br/>{{Study_Title_EN}})  
   - Study ID: {{Study_ID}}  
   - Study Director: {{Study_Director}}  
   - Test Facility: {{Facility_Name}}<br/>Address: {{Facility_Address}}<br/>Phone: {{Facility_Phone}}  
   - Sponsor: {{Sponsor_Name}}<br/>Address: {{Sponsor_Address}}  
   - Report Issue Date: {{Report_Sign_Date}}  (ISO: {{Report_Sign_Date_ISO}})  
   - Total Pages: {{Total_Pages}}  
   - Revision/Discrepancy Log: {{Discrepancy_Log_Entry}}  

4) 分支逻辑（缺项处理）
   - 若某字段在Protocol与Report中均缺失：停止生成模板并输出单行：该章节在原始材料中无明确数据支持  
   - 若Report缺失但Protocol存在：使用Protocol值填充并在{{Discrepancy_Log}}中记录“使用Protocol值—待确认”。  
   - 若Report存在但Protocol缺失：使用Report值并在日志中标注“仅Report记录”。  

5) 审计痕迹（写入要求）
   - 每次填充/修改均应记录操作人、操作时间、来源文件（文件名/页码）与理由，存入{{Metadata_Audit}}。
