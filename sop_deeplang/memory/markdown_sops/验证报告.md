## 一、核心填写规则
1. **静态文本复制原则**：验证报告的基本信息（如验证试验名称、编号、负责人、委托方及试验机构）必须直接从验证方案（VP）中逐字复制，确保无任何修改或省略。禁止添加未在VP中出现的描述。
2. **动态数据处理原则**：若VP中无额外实验数据或偏离记录，该章节仅作为标题页声明，无需引入计算或变量提取。若存在后续章节数据，该部分仅作引言性概述，不涉及具体数值。
3. **修约规则**：所有数值（如日期、编号）保留完整原貌，无需修约。文本描述统一采用正式商务语气，避免缩写或口语化表达。
4. **合规性底线**：报告开头必须包含完整试验标题，确保与VP一致。若VP为空或仅含声明，下游AI须输出标准化引言模板，声明“本报告基于验证方案执行，无偏离记录”。
5. **分支逻辑**：若检测到VP与报告间差异（如地址格式调整），优先采用VP原文；无差异时，直接复制。
