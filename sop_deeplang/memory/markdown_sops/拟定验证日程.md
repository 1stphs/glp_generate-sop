## 一、核心填写规则

1. **日程信息提取**：从验证方案（VP）中直接提取批准日期、实验开始日期、实验结束日期及报告初稿完成时间，确保日期格式统一为“YYYY年MM月DD日”。数据源为VP的“拟定验证日程”章节，无需额外计算或折算。
2. **变动处理逻辑**：若实际日程发生变动，记录为“注释”项，引用VP中预定义的说明（如“时间表为预估，如日程安排发生变动，无需更新方案或额外准备补充方案进行偏离解释”），不引入新判定标准；若无变动，直接声明“日程按方案执行，无偏离”。
3. **完整性核对**：验证报告（VR）中日程信息必须覆盖所有四个关键节点（批准日期、开始日期、结束日期、报告完成时间），若VP中任一节点缺失，标记为“待补充”并追溯数据源；针对低信息量场景（如无实际数据），默认使用VP静态文本生成声明式描述。
4. **无偏离分支**：若报告期内无日程变动，输出合规声明；若有变动，添加分支描述“实际日程调整为{{Adjusted_Date}}，符合VP预估注释，无需偏离解释”。
5. **通用化原则**：禁止使用具体日期值，所有输出使用占位符如{{Approval_Date}}；确保逻辑闭环，不依赖外部数据源。

**撰写指令**：  
从VP“拟定验证日程”章节提取以下信息：批准日期[{{Approval_Date}}]、实验开始日期[{{Start_Date}}]、实验结束日期[{{End_Date}}]、拟定报告初稿完成时间[{{Report_Completion_Date}}]。若日程无变动，填充模板A；若有变动，填充模板B并引用VP注释。输出格式为表格或列表，确保商务语气正式、简洁。
