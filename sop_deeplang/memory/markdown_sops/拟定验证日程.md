## 一、核心填写规则
1. **日程信息提取与一致性核对**：从验证方案（Protocol）中直接提取批准日期、实验开始日期、实验结束日期及报告初稿完成时间，确保日期格式统一为“YYYY年MM月DD日”。若方案中存在注释，需完整复制至报告中，作为日程灵活性的说明。
2. **无变动原则**：若实际日程与方案预估一致，直接声明“无偏离”；若发生变动，仅在注释中说明原因，无需修改方案或额外补充文件。统计参数（如日期差计算）修约至整数天，不涉及小数精度。
3. **数据源优先级**：优先使用方案原文日期；若报告阶段发现实际日期变动，记录实际值并标注“实际完成日期：{{Actual_Date}}（原预估：{{Planned_Date}}）”。
4. **逻辑闭环**：确保所有日期形成时间线（批准 → 开始 → 结束 → 报告），总时长计算公式：{{End_Date}} - {{Start_Date}}，修约至整数天。若无数据或无变动，输出标准声明“日程按方案执行，无偏离”。

## 二、通用标准化模板
**拟定验证日程**

**方案批准日期**：{{Approval_Date}}  
**实验开始日期**：{{Start_Date}}  
**实验结束日期**：{{End_Date}}  
**拟定报告初稿完成时间**：{{Draft_Completion_Date}}  

**注释**：{{Schedule_Notes}}（时间表为预估，如日程安排发生变动，无需更新方案或额外准备补充方案进行偏离解释。若有实际变动，记录为：实际{{Changed_Event}}日期：{{Actual_Date}}，原因：{{Change_Reason}}。总实验时长：{{Duration_Days}}天。）

**填写指令**：  
1. 从验证方案附件提取{{Approval_Date}}、{{Start_Date}}、{{End_Date}}、{{Draft_Completion_Date}}和{{Schedule_Notes}}。  
2. 计算时长：{{Duration_Days}} = ({{End_Date}} - {{Start_Date}}) 修约至整数。  
3. 若无变动，插入声明：“日程严格按照方案执行，无任何偏离。”  
4. 验证日期逻辑：确保{{Approval_Date}} ≤ {{Start_Date}} ≤ {{End_Date}} ≤ {{Draft_Completion_Date}}；若不符合，标记为“需核对”。
