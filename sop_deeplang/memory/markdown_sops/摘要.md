第一部分：核心准则
- 定义（必须遵守的“法律”）：
  1. 报告结论必须明确包含：{{Species}}、{{Anticoagulant}}、{{Analyte_Name}}、{{Analyte_Form}}（例如“游离型”）、{{Analyte_Strands}}（如“反义链和正义链”）和用于判定的方法学证据引用（{{Acceptance_Criteria_Ref}} / {{Supporting_Tables}}）。
  2. 严格区分静态文本与动态数据：
     - 静态文本（可直接复用的句式模板）：由本SOP模板提供的句子骨架（见第二部分）。
     - 动态数据（必须从原始记录或验证汇总表提取）：所有数值型指标、判定结果、表/附件编号、日期和样本描述（使用占位符表示，如{{Mean}}、{{CV}}、{{Stability_Result}}等）。
  3. 判定逻辑必须以验证方案或验证报告中的接受标准为准（{{Acceptance_Criteria_Ref}}）。结论仅在所有强制接受项目均满足时使用“符合……要求，可用于……”表述；若有任一强制项目不符合，必须使用不符合分支文字并列明限制与后续动作。
  4. 报告中所有数值的显示与计算保留位数由占位符控制（{{Decimal_Places}}），不得在模板中硬编码位数。
  5. 引用支持证据时必须逐项列出对应的表/附件编号：例如“见方法学验证汇总表（{{Summary_Table_Path}}）、稳定性数据（{{Stability_Table_Path}}）和原始记录（{{Raw_Data_Location}}）”。

第二部分：通用模板
- 数据提取（输入）：
  1. 从验证报告及原始记录提取以下字段并赋值：
     - {{Species}}、{{Anticoagulant}}、{{Analyte_Name}}、{{Analyte_Form}}、{{Analyte_Strands}}
     - 验证汇总表路径：{{Summary_Table_Path}}（包含{{Mean}}、{{CV}}、{{Accuracy}}、{{LLOQ}}、{{Stability_Result}}等列）
     - 接受标准引用：{{Acceptance_Criteria_Ref}}
     - 支持附件列表：{{Supporting_Tables}}、{{Raw_Data_Location}}
- 判定计算（逻辑步骤，原子化）：
  1. 对于验证表中每一项接受标准（列举为 i = 1..N）：
     - 取出实际结果 value_i = Summary_Table[i]
     - 取出对应接受阈值 criteria_i = Acceptance_Criteria_Ref[i]
     - 计算判定 pass_i = (value_i 满足 criteria_i) 或者按方案中规定的比较关系（例如绝对差、相对偏差等）；布尔结果保留为 {{pass_i}}。
     - 所有数值按 {{Decimal_Places}} 四舍五入后用于报告显示（保留位数通过占位符控制）。
  2. 聚合判定：{{Meets_Criteria}} = AND(pass_1, pass_2, …, pass_N)
- 文本生成（输出句式模板）：
  - 若 {{Meets_Criteria}} = true：
    模板句：
    “方法学验证结果表明，该方法符合生物样品测定要求，可用于 {{Species}}{{Anticoagulant}} 血浆样品中 {{Analyte_Name}}（{{Analyte_Form}}）{{Analyte_Strands}} 浓度的测定。支持证据见：{{Summary_Table_Path}}、{{Supporting_Tables}}。”
  - 若 {{Meets_Criteria}} = false：
    模板句：
    “方法学验证结果表明，该方法未完全满足全部预定接受标准（参见 {{Acceptance_Criteria_Ref}}），在以下方面不满足：{{Nonconformity_List}}。因此，本方法在{{Limitation_Description}}情形下不得用于定量测定；拟采取的纠正/再验证措施：{{Corrective_Action_Plan}}（详见附件：{{Revalidation_Plan_Location}}）。支持证据见：{{Summary_Table_Path}}、{{Supporting_Tables}}。”
- 引用与追溯（必填项）：
  1. 在结论句后必须追加至少一条支持证据引用，格式使用占位符列明表/附件编号：例如“支持证据见：{{Summary_Table_Path}}、{{Stability_Table_Path}}、原始记录（{{Raw_Data_Location}}）”。
- 失败分支要求（必备动作）：
  1. 若不符合，必须填充：{{Nonconformity_List}}、根本原因分析 {{Root_Cause}}、纠正措施 {{Corrective_Action_Plan}}、再验证时间表 {{Revalidation_Timeline}}。
- 输出格式规则：
  1. 单段结论，不超过三句，句内使用占位符替换后的完整信息。
  2. 所有动态数值按 {{Decimal_Places}} 显示并引用来源表格行列。
  3. 模板仅输出证据引用，不在正文中重复完整数值表（数值应在附表中呈现）。

第三部分：报告示例
方法学验证结果表明，该方法符合生物样品测定要求，可用于 SD大鼠EDTA-K2血浆样品 中 BPR-30160639（游离型）反义链和正义链 浓度的测定。
