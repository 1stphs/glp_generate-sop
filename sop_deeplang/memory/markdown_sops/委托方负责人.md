### **第一部分：核心准则**
1. **合规性底线**：严格遵守FDA 21 CFR Part 11/GLP要求，确保报告中所有数据追溯至原始记录（协议附件），计算过程透明、可审计。协议字符数（静态文本）直接复制，报告字符数从实际测量记录提取。
2. **二元差异定位**：静态文本（如协议字符数“66”）直接从验证协议（VP）复制；动态数据（如报告字符数“95”）从验证报告（VR）原始记录提取，不得篡改。
3. **通用化计算逻辑**：字符数计算为“实际报告文档总字符数（含空格）”，保留整数位，无修约。比较逻辑：报告字符 > 协议字符时，描述为“满足设计要求”；否则，触发不符合分支（“不符合设计要求，详见偏差记录”）。
4. **分支处理**：若报告字符数 < 协议字符数，必须在报告中明确记录偏差原因、影响评估，并引用纠正措施CAPA编号。
5. **数据流转**：从VP提取“{{Protocol_Chars}}”；从VR原始记录（如Word统计或附件截图）提取“{{Report_Chars}}”；描述句式固定为“验证报告字符数{{Report_Chars}}字符（协议{{Protocol_Chars}}字符）”。

### **第二部分：通用模板**
```
【执行指令】：
1. 从验证协议（VP）附件提取协议字符数，记为 {{Protocol_Chars}}（静态，直接复制）。
2. 从验证报告（VR）原始Word文档统计总字符数（含空格），记为 {{Report_Chars}}（动态，精确到整数）。
3. 比较：若 {{Report_Chars}} >= {{Protocol_Chars}}，则填充成功描述；否则，填充不符合描述并链接偏差表。

【填充模板】：
“验证报告字符数 {{Report_Chars}} 字符（协议 {{Protocol_Chars}} 字符），[判定结果]。”

【判定分支】：
- 符合：满足设计要求。
- 不符合：不符合设计要求，详见偏差记录 [CAPA_ID]。

【输出排版】：Arial 11号字体，左对齐，单倍行距，后接数据来源引用（如“数据来源：VR附件1，Word统计工具”）。
```
