### **第一部分：核心准则**
* 鉴于输入的验证方案 (Protocol)、GLP 报告 (Report) 和上一版本 SOP 均为空，本章节无具体实验数据、接受标准或差异逻辑可萃取。撰写时必须遵守 GLP 声明式原则：仅在无偏离或无数据时，使用简洁的声明句确认合规性，避免任何假设或编造内容。判定逻辑为二元（有数据时扩展描述，无数据时固定声明）。若后续输入数据，需补充取数-计算-对比流程，但当前仅限声明输出。保留位数默认为方案中未指定时的标准（例如，百分比保留1位小数）。

### **第二部分：通用模板**
* 执行以下脚本指令生成报告片段：  
  1. 检查附件或原始记录中是否有[章节名称]相关数据（如日期、结果值）。  
  2. 若无数据或确认无偏离，填入模板：  
     “本章节验证结果符合[方案引用或GLP标准]，无偏离记录。测试于{{Test_Date}}完成，所有操作按批准的验证方案执行。”  
  3. 若有数据，扩展为：从[数据源]提取{{Key_Value}}，计算{{Calculation_Step}}（例如，均值=SUM({{Values}})/N，CV%=({{SD}}/{{Mean}})*100），对比[接受标准]（例如，{{Limit}}内），输出判定：{{Compliance_Statement}}（符合/不符合）。  
  4. 结束语固定：结果支持方案有效性，无未记录偏差。
