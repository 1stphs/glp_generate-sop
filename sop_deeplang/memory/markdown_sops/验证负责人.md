### **第一部分：核心准则**
1. **数据来源优先级**：所有数据必须从原始记录（如 Excel 附件、仪器报告）中提取，不得凭空假设或直接复制协议中的示例值。
2. **合规性判定逻辑**：字符数必须严格符合协议要求（{{Protocol_Chars}}）。若报告字符数超出或不足，需在描述中明确说明偏差原因并评估对验证目的的影响。
3. **动态描述规则**：
   - **符合情况**：使用“符合”、“满足”等正面肯定词，并列出实际字符数。
   - **不符合情况**：使用“未符合”、“超出”等中性描述词，记录实际偏差值，并触发纠正措施说明。
4. **修约与精度**：字符数以整数修约，描述中保留实际计数完整值（如“94字符”），计算偏差时保留1位小数（公式：{{Report_Chars}} - {{Protocol_Chars}}）。
5. **GLP追溯性**：每段描述必须包含数据来源标识（如“附件X，Sheet1”）和计算时间戳，确保可审计。

### **第二部分：通用模板**
```
【执行指令】：
1. 从协议正文提取目标字符数：{{Protocol_Chars}}（来源：协议第X页）。
2. 从报告正文统计实际字符数：{{Report_Chars}}（来源：报告第X页，使用字符计数工具，如Word统计功能）。
3. 计算偏差：Deviation = {{Report_Chars}} - {{Protocol_Chars}}。
4. 判定并填充模板：

【通用描述模板】：
"根据验证协议要求，{{章节名称}}章节字符数应为{{Protocol_Chars}}字符。实际报告中，该章节字符数为{{Report_Chars}}字符，{{判定结果}}协议要求（偏差{{Deviation}}字符）。数据来源于[附件X，Sheet1]。"

【判定结果分支】：
- 若 |Deviation| ≤ 5：{{判定结果}} = "符合"
- 若 Deviation > 5："超出{{Protocol_Chars}}字符"
- 若 Deviation < -5："不足{{Protocol_Chars}}字符"
- 若不符合：追加"建议优化：{{优化建议，如'精简冗余描述'}}"
```

根据验证协议要求，章节字符数应为58字符。实际报告中，该章节字符数为94字符，未符合协议要求（偏差36字符）。数据来源于报告正文，使用Microsoft Word字符统计功能（计数日期：2025-04-17）。建议优化：精简方法描述中的重复性语句。
