第一部分：核心准则
1. 必须记录并核验三项必填联系信息字段：姓名、电话、电子邮件。任何一项缺失即判为信息不完整并记录不一致原因。
2. 静态字段（姓名）与动态字段（电话、电子邮件）均应从原始来源逐字比对；比较前对文本进行统一化处理（去除 HTML 标签、首尾空白、全角/半角统一）。
3. 比对策略：
   - 优先将 Protocol 中对应字段作为“方案来源”，将 Report 中对应字段作为“呈现来源”；若方案缺失则反向采用报告为基准并标注为“方案缺失”。
   - 要求字符串精确一致；若存在差异，记录“方案值 / 报告值 / 差异类型（拼写/格式/缺失）”。
4. 格式校验：
   - 电话：应匹配通用国际电话号码正则（例如：^\+?[0-9\-\s()]{6,}$），不允许含有字母；如含扩展号，须使用明确分隔符并原样记录。
   - 电子邮件：应匹配标准邮件正则（例如：^[^\s@]+@[^\s@]+\.[^\s@]+$）。
5. 追溯与出处：
   - 每次提取必须记录数据来源文件名或段落标识、提取人/工具、提取日期时间。
6. 不一致处置逻辑：
   - 若字段存在且格式合法但值不一致：标注为“值不一致”，并触发人工复核步骤（记录复核结果与时间）。
   - 若字段格式不合法：标注“格式异常”，并建议按原始记录保留原文同时附注格式问题。
7. 输出要求：最终用于报告的段落应包含经核验的姓名、电话、电子邮件三项，且在报告旁注明数据来源与校验状态（一致 / 不一致 / 缺失 / 格式异常）。

第二部分：通用模板
步骤指令（可直接由 AI Generator 执行）：
1. 从 Protocol 文档中定位“验证负责人”节，抽取字段：
   - {{Protocol_Name_Source}} -> {{Name}}
   - {{Protocol_Phone_Source}} -> {{Phone}}
   - {{Protocol_Email_Source}} -> {{Email}}
   并记录 {{Protocol_File}} 与 {{Protocol_Extraction_Datetime}}。
2. 从 Report 文档中定位对应呈现段落，抽取字段：
   - {{Report_Name_Source}} -> {{Name_report}}
   - {{Report_Phone_Source}} -> {{Phone_report}}
   - {{Report_Email_Source}} -> {{Email_report}}
   并记录 {{Report_File}} 与 {{Report_Extraction_Datetime}}。
3. 归一化处理（作用于所有抽取值）：
   - 去除 HTML 标签与多余空白，统一全半角，修剪首尾空格。
4. 字段存在性检查：
   - 若三项均为空：输出单行文本 "该章节在原始材料中无明确数据支持" 并停止执行。
5. 格式校验：
   - 验证 {{Phone}} 与 {{Phone_report}} 是否匹配电话正则；验证 {{Email}} 与 {{Email_report}} 是否匹配邮件正则；记录校验结果字段 {{Phone_Format_OK}}/{{Email_Format_OK}}。
6. 值一致性比对：
   - 若 {{Name}} 等于 {{Name_report}}（精确字符串匹配）则标注 Name_Status = 一致， 否则标注 Name_Status = 不一致，并记录差异为 {{Protocol_Name_Source}} / {{Report_Name_Source}}。
   - 对 Phone 与 Email 同上，分别产生 Phone_Status 与 Email_Status。
7. 输出构建（用于填充至最终报告章节的模板，使用占位符）：
   - 输出文本（占位符替换规则）：
     <p>姓名：{{Resolved_Name}}</p>
     <p>电话：{{Resolved_Phone}}</p>
     <p>电子邮件：{{Resolved_Email}}</p>
   - 其中 {{Resolved_*}} 优先取一致且格式合格的值；若不一致但协议有值且格式合格，则以协议值为首选并在旁注 "{{Field}}（报告值：{{Report_Value}}；比对状态：{{Status}}）"；若协议缺失则以报告值并注记“方案缺失”。
8. 差异与审计记录：
   - 生成一条审计记录 JSON 结构包含：{ "field": ..., "protocol_value": ..., "report_value": ..., "status": ..., "protocol_source": ..., "report_source": ..., "extracted_at": ... } 并保存到指定路径 {{Audit_Output_Path}}。
9. 人工复核触发：
   - 若任一字段 Status != 一致 或 格式校验失败，则在输出中附加复核动作项并收集复核人、复核时间与复核结论字段（占位符 {{Reviewer}}/{{Review_Datetime}}/{{Review_Conclusion}}）。
