### **第一部分：核心准则**
* 鉴于输入的验证方案 (Protocol) 和 GLP 报告 (Report) 均为空，本章节撰写时必须遵守声明式原则：仅生成无偏离或无数据支持的标准化声明。 
* 核心逻辑：如果无具体数据或偏离，报告中不得引入任何假设性数值或计算；直接声明“符合方案要求”或“无偏离记录”，并确保表述中立、简洁。
* 分支处理：若未来输入有数据，优先提取实际值进行描述；否则，默认无偏离路径。
* 通用化要求：使用占位符如 `[{{Declaration_Type}}]` 表示声明内容，避免任何具体细节。

### **第二部分：通用模板**
* **指令脚本**：  
  1. 检查验证方案 (Protocol) 是否为空：如果是，跳至步骤 3。  
  2. 从 Protocol 中提取关键要求（如接受标准），若无，则使用默认声明“无具体要求”。从报告 (Report) 中提取任何可用数据或偏离记录；若无数据，标记为“无偏离”。  
  3. 填充模板：`本章节验证结果显示 [{{Declaration_Type}}]（例如：无偏离/符合要求），详见附件 [{{Attachment_Reference}}]`。  
  4. 若有数据，计算并对比（如适用）：提取 [{{Data_Source}}]，应用 [{{Acceptance_Criteria}}] 进行判定，输出“符合/不符合”并说明原因。  
  5. 结束：确保输出不超过 50 字，保持 GLP 合规语气。
