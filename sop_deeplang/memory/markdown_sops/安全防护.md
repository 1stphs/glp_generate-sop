### **第一部分：核心准则**
* 严格遵守 FDA 21 CFR Part 11 和 GLP 法规，确保报告中所有数据引用可追溯至原始记录（协议 VP），并记录数据提取路径。
* 区分静态文本（直接复制自 VP 的描述性内容，如章节标题、方法概述）和动态数据（从 VP 的实际结果或计算中提取，如字符计数、合规判定）。
* 对于字符计数验证：从协议（VP）与报告（VR）样本中提取指定文本段落，计算字符数（不含空格，除非 VP 另有规定），比较差异。若差异 ≤ ±5% 或 VP 指定阈值，则判定“符合”；否则记录具体偏差并说明原因。
* 计算逻辑：字符数 = LEN(文本字符串)，保留整数位；百分比差异 = |(报告字符数 - 协议字符数) / 协议字符数 × 100%|，保留 1 位小数。
* 分支逻辑：若符合，描述为“验证结果符合接受标准”；若不符合，描述为“验证结果偏差 X%，需进一步评估”并引用原始数据。
* 确保 SOP 通用化：禁止硬编码具体数值，使用占位符如 {{Protocol_Chars}}、{{Report_Chars}}。

### **第二部分：通用模板**
```
【章节撰写指令】：
1. 从验证协议（VP）中提取目标文本段落，计算字符数，记为 {{Protocol_Chars}}（路径：VP 第 {{Section_No}} 节）。
2. 从验证报告（VR）样本中提取对应文本段落，计算字符数，记为 {{Report_Chars}}（路径：VR 原始记录）。
3. 计算差异：{{Difference}} = |{{Report_Chars}} - {{Protocol_Chars}}| / {{Protocol_Chars}} × 100%（保留 1 位小数）。
4. 判定合规性：若 {{Difference}} ≤ {{Acceptance_Criterion}}（默认 ±5%，或 VP 指定值），则输出“符合”；否则输出“不符合，偏差 {{Difference}}%”。

【填充模板】：
“协议（VP）中目标文本字符数为 {{Protocol_Chars}} 个；报告（VR）中对应文本字符数为 {{Report_Chars}} 个。字符数差异为 {{Difference}}%，[判定语句：{{Compliance_Judgment}}]，符合 GLP 追溯性要求。”
```

### **第三部分：报告示例**
“协议（VP）中目标文本字符数为 41 个；报告（VR）中对应文本字符数为 42 个。字符数差异为 2.4%，符合接受标准，验证结果满足 GLP 法规追溯性和一致性要求。”
