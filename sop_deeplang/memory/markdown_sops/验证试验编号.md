第一部分：核心准则
- 定义：本章节仅核对并固定“验证试验编号”（标签为“验证试验编号”或等效中文/英文表述）这一项静态标签下的动态数据项（试验编号字符串）。
- 静态文本 vs 动态数据：静态文本为字段名称“验证试验编号”；动态数据为字段值（例如：NS25315BV01）。
- 校验规则：协议中提取的试验编号与报告中提取的试验编号必须在经标准化处理后完全一致（逐字符匹配）。标准化处理包括：前后空白裁剪、统一大小写（记录是否改变）、消除不可见控制字符，但不得删除或改写数字/字母顺序。
- 不符处置：若不一致，记录比较结果并生成差异记录（Deviation/Discrepancy），保留原始来源截屏/摘录，通知质量/项目负责人并在SOP指定的偏差流程中跟踪直至关闭。不得在最终报告中擅自替换任一来源编号而不留变更记录。
- 取证要求：比对应附带来源定位（文件名、页码/段落、提取时间、提取人）和原始截图或文本片段作为证据链。

第二部分：通用模板
- 输入源定义：
  - {{Protocol_File}}：包含“验证试验编号”的验证方案文档或字段位置（示例占位：Protocol -> section "验证试验编号"）。
  - {{Report_File}}：包含报告中“验证试验编号”的报告文档或字段位置（示例占位：Report -> first occurrence of "验证试验编号"）。
- 提取步骤：
  1. 从 {{Protocol_File}} 精确提取字段值，存为 {{Protocol_Study_ID}}。并记录来源元数据：{{Protocol_File}}, {{Protocol_Page_or_Section}}, {{Extract_Time}}, {{Extractor}}。
  2. 从 {{Report_File}} 精确提取字段值，存为 {{Report_Study_ID}}。并记录来源元数据：{{Report_File}}, {{Report_Page_or_Location}}, {{Extract_Time}}, {{Extractor}}。
- 标准化步骤（必须逐步执行并记录是否有变化）：
  1. 对 {{Protocol_Study_ID}} 和 {{Report_Study_ID}} 做 trim()（去除前后空白）。
  2. 记录原始大小写，按需转换为统一大小写用于比较（记录转换方式）：{{Normalized_Protocol_ID}} = upper({{Protocol_Study_ID}}) 或 lower(...)。
  3. 去除不可见控制字符但不改变可见字符顺序。
- 比较逻辑：
  - 如果 {{Normalized_Protocol_ID}} == {{Normalized_Report_ID}}：
    - 生成合规结论句模板： "验证试验编号：{{Protocol_Study_ID}} — 与验证方案一致（已比对来源：{{Protocol_File}} 第{{Protocol_Page_or_Section}}页；报告来源：{{Report_File}} {{Report_Page_or_Location}}）。"
    - 将比对记录存入审核表（字段：Protocol_ID, Report_ID, Normalization_Method, Extract_Times, Extractors, Evidence_Files）。
  - 否则（不一致）：
    - 生成不一致记录模板： "验证试验编号不一致 — 协议记录：{{Protocol_Study_ID}} (来源：{{Protocol_File}} 第{{Protocol_Page_or_Section}}页)；报告记录：{{Report_Study_ID}} (来源：{{Report_File}} {{Report_Page_or_Location}})。已提交偏差：{{Deviation_ID}}；已通知：{{Notified_Personnel}}；证据附后：{{Evidence_Files}}。"
    - 立即创建偏差单（占位 {{Deviation_ID}}），并将所有原始截屏/摘录与上述比较表一并上传。
- 输出位置与格式化（用于生成最终报告段落）：
  - 成功模板（占位）： "验证试验编号：{{Protocol_Study_ID}} — 与验证方案一致（见来源：{{Protocol_File}} / {{Report_File}}）。"
  - 不一致模板（占位见上“否则”段）。
- 审计痕迹要求：比对结果、标准化动作、提取时间、提取人、证据文件名必须写入项目核对表并由审核人签名（占位字段：{{Reviewer}}, {{Review_Date}}）。
- 失败分支处理时间窗口与通知占位（不得固化具体时限）：使用占位符 {{Notification_Hours}} 和 {{Deviation_Resolution_Target}} 由项目管理或QA填写。
