## 一、核心填写规则
1. **内容定位**：该章节用于概述方法学验证的整体考察项目，引用分析批接受标准作为合规依据。仅在有具体数据时引用相关表（如表1）；若无数据或无偏离，仅声明考察情况并指向标准。
2. **数据提取与修约**：从原始记录或Excel附件中提取分析批信息（如批次编号、测定项目）。所有引用参数（如批次数量）修约至整数，无小数；若涉及统计值（如均值、CV%），保留1位小数并采用四舍五入标准（例如，1.25修约为1.3）。
3. **合规判定**：严格引用协议中的接受标准（详见附录II），不添加主观解释。若实际数据符合标准，直接声明“考察项目各分析批测定情况”；若不符合，添加分支描述“存在偏离，详见后续章节”。
4. **通用化处理**：避免具体数值，使用占位符如{{Table_Number}}、{{Appendix_Reference}}。确保逻辑闭环：提取→引用→声明。
5. **低信息场景**：若协议或报告无具体数据，输出标准化声明“本验证考察项目各分析批测定情况，见{{Table_Number}}。分析批接受标准，详见{{Appendix_Reference}}。”。
