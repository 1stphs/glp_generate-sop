### **第一部分：核心准则**
* **定义**：该章节撰写时必须死守的"法律"。

1. **数据来源与完整性**：从表13_Dilution（稀释可靠性）中提取所有指定维度数据，包括检测时间、检测批次编号、DQC（稀离倍数20倍）的检测时间、检测批次编号、DQC值（ng/mL）、准确度偏差%（个体值）、平均值、SD、CV（%）、准确度（%）和n值。确保数据覆盖游离型与盐型的折算逻辑：若数据为盐型浓度，使用折算因子（游离型浓度 = 盐型浓度 × 分子量比，例如盐型分子量/游离型分子量）转换为游离型等效值，并在准则中注明因子定义（如基于化合物特定分子量比，默认为1.0若无指定）。
2. **接受标准**：准确度偏差须≤±15.0%（个体值），准确度（%）≥85.0%，CV（%）≤15.0%。计算步骤原子化：(1) 个体准确度偏差% = [(测定值 - 理论值)/理论值] × 100；(2) 平均准确度% = (平均测定值 / 理论值) × 100，修约至小数点后一位；(3) CV% = (SD / 平均值) × 100，修约至小数点后一位。若不符合，触发分支描述“结果不符合接受标准，详见偏差记录”。
3. **判定逻辑**：基于计算结果，生成闭环判定——若所有指标合规，表述“稀释不影响准确性”；否则，列出具体超标指标并建议重复验证。通用化处理：使用{{Dilution_Factor}}（如20倍）、{{Nominal_Concentration}}（如500 ng/mL）、{{Accuracy_Deviation}}（个体偏差%）、{{Mean_Accuracy}}（平均准确度%）等占位符，避免具体数值。
4. **分支处理**：无数据时，默认表述“按验证方案执行，无偏离”；不符合时，添加“潜在影响：{{Issue_Description}}，需{{Remediation_Step}}”。
