第一部分：核心准则
1. 定义与二元区分：方案中列示的“主要操作人员”为静态文本（Protocol source，字段：{{Protocol_Primary_Operators}}）；实际在研究中执行操作的人员为动态数据（Raw records，字段：{{Actual_Primary_Operators}}）。报告中必须明确数据来源并能追溯到相应原始记录。  
2. 呈现规则：报告“主要操作人员”段落应反映方案列示内容；若实际执行人员与方案列示一致，仅在报告中列出方案中的名称并标注“与方案一致”（可选短句）；若存在差异，报告必须同时列示方案列示与实际执行两组名称，并包含标准说明语句（见模板）。  
3. 人员变动处理红线：遵循方案注释——“如以上人员发生变动，无需更新方案或额外准备补充方案进行偏离解释”。在报告中引用该注释作为处理理由，但仍须在可追溯的研究记录中记录变动（姓名、变动日期、涉及的任务/阶段、指向原始记录的索引）。  
4. 可追溯性要求（最小项）：所有在报告中出现的姓名必须能对应到至少一种原始记录（如签到表、样品操作记录、仪器运行日志等），并在交付包中给出引用索引（例如：附件编号/页码/电子文件名）。  

第二部分：通用模板
操作步骤（可直接由AI Generator执行）：
1) 提取：从Protocol文件中读取“主要操作人员”字段 -> 变量{{Protocol_Primary_Operators}}。  
2) 提取：从原始研究记录（签到/操作记录/样品登记等，路径由{{Raw_Record_Paths}}给出）汇总实际执行人员 -> 变量{{Actual_Primary_Operators}}，并同时提取对应日期/任务的索引 -> 变量{{Actual_Indices}}。  
3) 对比判定：
   - 如果 {{Actual_Primary_Operators}} 等于 {{Protocol_Primary_Operators}}（字符串集合完全匹配），设置报告段落变量：{{Report_Primary_Operators}} = {{Protocol_Primary_Operators}}；且在内部注记{{Trace_Indices}} = {{Protocol_Source}}。  
   - 如果不一致，设置报告段落变量：{{Report_Primary_Operators}} = 同时包含 {{Protocol_Primary_Operators}} 与 {{Actual_Primary_Operators}}；并准备变动说明文本：{{Change_Statement}} = "根据方案注释，人员变动无需更新方案或额外准备补充方案进行偏离解释。"  
4) 生成报告文本（占位符形式，禁止出现硬编码具体数字或未授权文本）：
   - 若一致：输出字段 "主要操作人员：{{Report_Primary_Operators}}"；可附注："（与方案列示一致）"（可选）。  
   - 若不一致：输出两个子项：
       a) "方案列示主要操作人员：{{Protocol_Primary_Operators}}"  
       b) "实际参与本研究执行的主要操作人员：{{Actual_Primary_Operators}}；参见原始记录索引：{{Actual_Indices}}"  
       c) 紧接标准说明行： "{{Change_Statement}}"  
5) 附索引输出：在报告附件或注释处自动生成索引表格（占位符）：{{Trace_Indices}} -> 列出每一姓名对应的原始记录路径/文件名/页码。  
6) 输出位置：将上文生成的段落写入报告章节“主要操作人员”；将索引表写入报告附录/原始资料索引表。
