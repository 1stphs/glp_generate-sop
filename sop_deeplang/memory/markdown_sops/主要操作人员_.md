### **第一部分：核心准则**
* 本章节严格遵守FDA 21 CFR Part 11/GLP法规，确保所有数据完整性、审计追踪性和可追溯性。
* 字符统计必须基于协议（VP）和报告（VR）文件的实际文本内容，使用UTF-8编码进行计数，不包括文件头/尾标记或不可见字符。
* 差异计算公式：Δ = |报告字符数 - 协议字符数| / 协议字符数 × 100%，保留1位小数；接受标准：Δ ≤ 15.0%（若超出，需触发偏差调查并记录根因）。
* 数据来源优先级：从原始文件附件提取精确字符数，手动验证后记录；禁止估算或主观调整。
* 分支逻辑：
  * 符合：声明“字符数符合预定标准，无显著差异”。
  * 不符合：客观报告实际值、差异百分比，并引用GLP偏差管理流程（参考SOP-DEV-001）。
* 修约规则：字符数为整数，百分比保留1位小数（四舍五入）。

```
2. 从验证报告（VR）附件{{VR_File_ID}}提取总字符数：{{Report_Chars}} 字符（UTF-8编码）。

3. 计算字符差异：
   - 绝对差异：{{Absolute_Diff}} = |{{Report_Chars}} - {{Protocol_Chars}}|
   - 相对差异：{{Relative_Diff_%}} = ({{Absolute_Diff}} / {{Protocol_Chars}}) × 100% = {{Calculated_Relative_Diff}}%

4. 合规性判定：
   {{#if Relative_Diff_% <= 15.0}}
   该差异（{{Relative_Diff_%}}%）符合接受标准（≤ ±15.0%），确认文档内容完整性无显著偏差。
   {{else}}
   该差异（{{Relative_Diff_%}}%）超出接受标准（> ±15.0%），已记录为偏差（DEV-{{ID}}），根因分析见附件{{Deviation_Report}}。
   {{/if}}

数据已双人审核，审计追踪ID：{{Audit_Trail_ID}}。
```

### **第三部分：报告示例**
【章节标题：文档字符统计验证】

1. 从验证协议（VP）附件VP-52-001提取总字符数：52 字符（UTF-8编码）。

2. 从验证报告（VR）附件VR-06-001提取总字符数：6 字符（UTF-8编码）。

3. 计算字符差异：
   - 绝对差异：46 = |6 - 52|
   - 相对差异：88.5% = (46 / 52) × 100%

4. 合规性判定：
   该差异（88.5%）超出接受标准（> ±15.0%），已记录为偏差（DEV-2024-001），根因分析见附件Deviation-52-06.pdf。

数据已双人审核，审计追踪ID：AT-2024-0526-001。
