# Complexity Analysis Skill v1.0

## 职责
分析章节的复杂度并确定路由路径

## 复杂度分类

### Simple（简单章节）
**定义**：纯格式化内容，无需操作步骤或复杂判断

**特征**：
- 缩略词表、术语表
- 参考文献、版本历史
- 目录、附录
- 致谢、签字页
- 内容少于 200 字
- 纯信息展示，无操作步骤

**路由**：`simple_path` → Writer → Format Verify → END

### Complex（复杂章节）
**定义**：需要统计分析、多步骤操作或复杂验证的章节

**特征**：
- 方法学验证
- 稳定性研究
- 系统适用性测试
- 数据分析与统计
- 批内/批间准确度及精密度
- 基质效应分析
- 回收率验证
- 包含"计算"、"统计"、"分析"、"验证"等关键词
- 内容超过 1000 字
- 有多个嵌套的子步骤

**路由**：`complex_path` → Writer → Simulator → Reviewer → (FAIL) → Analyzer → Curator → Writer (循环)

## 判断标准

### 优先级 1：章节名称匹配
**直接判定为 Simple 的章节**：
- 缩略词表
- 缩写
- 参考文献
- 版本历史
- 修订记录
- 目录
- 附录
- 致谢
- 术语
- 符号说明
- 单位说明

**直接判定为 Complex 的章节**：
- 方法学验证
- 稳定性研究
- 统计分析
- 数据分析
- 结果与讨论
- 质量控制
- 系统适用性
- 样品制备
- 色谱条件
- 质谱条件
- 定量分析
- 定性与定量

### 优先级 2：内容分析（当章节名称无法确定时）

**Simple 特征**：
- 内容 < 200 字
- 无实验步骤
- 纯格式化内容（表格、列表）
- 纯信息展示

**Complex 特征**：
- 内容 > 1000 字
- 包含以下任一关键词：
  - "计算"、"统计"、"验证"、"分析"
  - "数据"、"稳定"、"方法"、"质量"
  - "准确度"、"精密度"、"回收率"
  - "基质效应"、"选择性"、"特异性"
- 有多个子步骤（编号 1.1, 1.2, 1.3...）
- 需要参数计算（如 RSD、CV、AQL 等）

**其他情况**：如果不符合 Simple 或 Complex 的明确特征，根据内容性质判断：
- 有操作步骤且包含参数 → Complex
- 纯信息展示 → Simple

## 输出格式

```json
{
  "complexity": "simple|complex",
  "route": "simple_path|complex_path",
  "reasoning": "判断依据（50字以内）"
}
```

## 示例

### 示例 1：Simple 章节
**输入**：
```
章节名称：缩略词表
验证方案：本章节定义文档中使用的缩略词。
GLP 报告：缩略词表
```

**输出**：
```json
{
  "complexity": "simple",
  "route": "simple_path",
  "reasoning": "纯格式化内容，无操作步骤"
}
```

### 示例 2：Complex 章节
**输入**：
```
章节名称：方法学验证
验证方案：验证分析方法的有效性，包括准确度、精密度、专属性、检测限、定量限、线性、范围等参数。
GLP 报告：准确度要求±5%，精密度RSD<3%。
```

**输出**：
```json
{
  "complexity": "complex",
  "route": "complex_path",
  "reasoning": "需要统计分析，包含多个验证参数"
}
```

### 示例 3：需要判断的章节
**输入**：
```
章节名称：样品制备
验证方案：样品需要经过预处理，包括离心、过滤等步骤。离心条件：4°C，12000rpm，10分钟。
GLP 报告：样品制备：将样品在4°C下以12000rpm离心10分钟，然后通过0.22μm滤膜过滤。
```

**输出**：
```json
{
  "complexity": "complex",
  "route": "complex_path",
  "reasoning": "包含操作步骤和关键参数"
}
```

## 注意事项

1. **不要使用"标准"(standard)复杂度**：只输出 simple 或 complex
2. **优先参考章节名称**：如果章节名明确属于 Simple 或 Complex 列表，直接判定
3. **分析输入内容**：仔细阅读验证方案和 GLP 报告内容
4. **输出简洁的理由**：50 字以内，说明为什么这么判断
5. **格式要求**：只输出 JSON，不要任何额外文字

## 错误处理

如果无法确定复杂度，默认为 complex（更保守），并在 reasoning 中说明原因。
