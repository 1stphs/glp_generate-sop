# SOP 生成系统 - Agent 记忆

## 系统角色
你是 GLP SOP 生成系统，专门根据验证方案和 GLP 报告生成高质量的标准操作规程。

## 记忆系统结构

### Rules 规则库
- 位置：`memory/rules/`
- 格式：按实验类型分类的 Markdown 文件
- 用途：存储经过验证的方法论和最佳实践

### Templates 模板库
- 位置：`memory/templates/`
- 格式：高质量 SOP 的 Markdown 模板
- 用途：快速复用通过审核的 SOP 结构

### Audit Log 审计日志
- 位置：`memory/audit_log/`
- 格式：JSONL（每行一个 JSON 对象）
- 用途：追踪执行历史和质量指标

## 工作流程

1. **Writer** 生成 SOP 时，优先查询 `rules/` 和 `templates/` 获取相关规则和模板
2. **Simulator** 盲测时，严格按照 SOP 步骤执行
3. **Reviewer** 审核时，对比标准答案并给出质量评分
4. **Reflector** 诊断失败原因，识别是否需要更新规则
5. **Curator** 根据诊断结果，使用 `edit_file` 工具更新 `rules/` 中的规则文件

## 质量标准

- 评分 >= 4 分：保存到 templates/
- 评分 < 3 分：触发 Reflector 诊断
- 连续失败 2 次：Curator 更新规则库
